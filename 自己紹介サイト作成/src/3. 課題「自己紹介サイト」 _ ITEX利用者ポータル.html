3. 課題「自己紹介サイト」

2025.06.24 2022.02.23

課題「自己紹介サイト」では、実業務を模した課題制作を行っていただきます。実際のWeb制作の現場で使用される形式に近い指示書やデザインファイルなどを使用し、コーディングを行っていただきます。

上記が終了したのち、別ページ「JavaScript課題」を実装し、「自己紹介サイト」提出ルールに則った過程を経たあと、ITSに提出してください。

お願い

本ポータルサイトでご案内している各スプレッドシートは閲覧専用です。
編集はできませんので、内容を加工・編集される場合は、お手数ですがExcel形式で保存（ダウンロード）またはご自身のドライブにコピーを作成し、貴オフィス内のPCでご利用ください。

想定されるシチュエーション

あなたはWeb制作会社「株式会社ディーキャリア」の新入社員です。
今回は、個人のクライアント「就労太郎さん」から「自己紹介ページ」を依頼され、研修を終えてから初めて、コーダーとしてコーディングを担当することになりました。

制作は下記のチームで行っています。

  • ディレクター：田中三郎さん（あなたの上司）
  • デザイナー：山口花子さん（あなたの同僚）
  • コーダー：あなた

現在はデザインまでが完了し、クライアントの就労太郎さんからOKが出ている状態です。

支給されるファイルについて

「就労太郎様案件コーダー共有事項」

  • 下記リンクが今回の案件のために「社内で使用するドキュメント」をイメージしたものです
  • 実際に社内の共有で使用されるドキュメントに近い体裁で作成されています（会社によって独自で様々な形式のドキュメントを使用するので、必ずしもこの形式というわけではありません）
「就労太郎様案件コーダー共有事項」を確認する

「就労太郎様案件コーダー共有事項」各シート解説

  • 案件概要：案件の概要が書かれています。デザインカンプ（後述）や使用画像のリンク、納品形式など、制作に必要な情報も記載されているため、ご留意ください。
  • 見出し：hタグの情報です。hタグはSEO対策に必要なため、このようにディレクターから指定されることがあります。
  • 社内コーディングルール：社内で定められている独自のコーディングルールです。このルールに従ってコーディングを行ってください。
  • 山口注釈：デザイナー山口さんががコーディングの際に気を付けて欲しいことを記載したメモです。こちらを守ってコーディングしてください。
  • アドバイス（田中）：新入社員であるあなたのためにディレクターの田中さんが記述したコーディングのアドバイスも入っています。（このようにアドバイスが個別でもらえることはあまり多くないのですが、こちらにかかれていることは課題の中の位置づけとしては「基礎的なスキルが身についているかのチェックのための要件であるが、要件として業務中に明示的に要求されることがないこと」になります）

デザインカンプについて

デザインカンプとは、デザインの完成形のデータのことを指します。

今回は社内のデザイナーがAdobe XDで作成したものを参照しながらコーディングをしていきます。

XDからのコーディングの仕方

情報の取得の仕方① 要素にフォーカスを当てる

「案件概要」に記述されているXDのリンクを開き、パスワードを入力すると、デザインが表示されます。真ん中のデザインが出ているエリアはブラウザのウィンドウサイズ（今回は1280px）を想定しており、エリア内をスクロールすることによって全体を見ることができます。

コーディングを行うための情報を取得するためには、右上に3つ並んでいるアイコンのうち、

↑をクリックします。

このモードでは、画面の上にマウスカーソルを当てたり、要素をクリックすることによって数値や色などの値を取得することができます。

カラーコードを取得したり、

テキストをコピーすることができます。

こういった機能を使用し、見た目の通りコーディングを行なっていきます。思った通りの値を取るためにカーソルの動かし方やクリックの仕方に工夫が必要ですが、要素同士の余白の長さなど、コーディングに必要な要素は全て参照することができます。

「山口注釈」などでXDよりも優先させて欲しいと書かれている要件についてはXDよりも優先させてください。

情報の取得の仕方② CSSプロパティ

左図のように、XDの右下にはCSSが記述されています。カラーコードや文字間隔など、必要のあるものはコピーして使用してください。

また、横幅等の数値も使用して問題ありませんが、このスタイルをそのままコピーした結果要件を満たさなくなる場合もありますので、機械的にコピーするのではなく、必要な要素を判別して使用してください。

特に、「top」や「left」などの位置を制御するプロパティは本課題では使用しないでください。これらはドキュメントの開始からの絶対値が機械的に計算されたもの（すべての要素をabsoluteで絶対値指定したもの）ですので、実務でもそのまま使用することは特殊な場合を除いてまずありません。また、opacity: 1;やletter-spacing: 0px; などのデフォルトの値や、指定する必要のないwidthやheightなど、記述しなくても結果が変わらないものは省いてください。

実装に使用するツール

テキストエディタ

Progateの学習とは違い、実際のWebサイトのコーディングは「テキストエディタ」を使用して行うことが一般的です。テキストエディタとは、文字（テキスト）を編集したり保存したりを行うツールです。省略して「エディタ」と呼ばれることもあります。単に「エディタ」と言う時にはテキストエディタを指していることが多いですが、テキスト以外のエディタ（グラフィックエディタ・バイナリエディタ）も存在していることを知っておいてください。

Visual Studio Code （VSCode）

訓練ではVisual Studio Code（VSCode、ぶいえすこーど、と呼ぶことが多いです）を使用いただきます。2015年にリリースされ、テキストエディタの中ではシェア一位となっている人気のエディタです。

これまで学習してこられた方の中には、別のエディタを使用してきた方もいらっしゃると思いますが、就職先の現場によってはプロジェクトで指定されたツールの使用が望まれる場合がありますので、そのための訓練という位置付けでVSCodeを使用していただくことを推奨します。

VSCodeは、基本的には事業所のPCにインストールされておりますが、もしも見当たらない場合などはオフィススタッフにお問い合わせください。VSCodeで実装を行い、拡張子「.html」で保存したファイルをダブルクリックすることによって、ブラウザでWebページとして確認することができるようになります。

拡張機能（プラグイン）について

VSCodeには、元々の機能を「拡張」し、便利にするための「拡張機能」がございます。使用に際しましては、できるだけサポートが整っていて更新頻度の高い、使用実績のあるプラグインを選択するようにしてください。原則として、下記を満たしている必要がございます。

  • ダウンロード数が他のプラグインと比べて少なく無いもの
  • 使用方法が記述されている日本語記事があること（あるいは、利用者様側で信頼に足りると判断できる英語記事が存在すること）
  • 訓練で使用する範疇における脆弱性についての指摘がされていないもの

W3C Markup Validation Service

使用方法

１．W3C Markup Validation Serviceに接続します。
　　https://validator.w3.org/#validate_by_upload
２．[ファイルを選択]から確認したいhtmlファイルを選択します。

３．[check]をクリックします。
４．結果を確認します。

結果確認について

・通知がある場合、「Error（エラー）」「Warning（警告）」「Info（情報）」が結果表示されます。
・Errorとあるもののみ修正をお願いいたします。
・WarningやInfoは対応不要です。
・Error/Warning/Infoのいずれも無く、”No errors or warnings to show.”と出力されていた場合はエラーが無いことを示します。

Error対応について

内容は英語表記ですが、文言を検索する等してどういったエラー内容なのかご自身でお調べください。
実務でもエラー内容を自分で調べる場面は多々ございますので、訓練と思ってご対応ください。また、どの要素で出ているエラーなのか確認し、該当する要素について概要を確認することを推奨します。
翻訳ツールは日本語として意味が通らない場合がございますのでご注意ください。内容の把握が難しい場合はRedmineにてご質問ください。
